    Req.Decoder behaviour (req v0.6.0-dev)

    View Source

    Behaviour for response body decoders.

    A decoder declares:

    • mime_formats/0 — the MIME format keys it handles (e.g. :json, :ndjson).
    • stream?/0 — whether the decoder supports streaming via :stream.
    • decode_eager/1 — decode a full buffered body (used when :stream is not set).
    • decode_start/0, decode_chunk/2, decode_finish/1 — chunk-state machine for streaming decoders. Required only when stream?/0 returns true.

    Req.Decode looks decoders up by format key (via the :decoders option) and dispatches based on the response content-type header.

    Eager-only decoder

    defmodule MyCSV do
  @behaviour Req.Decoder

  @impl true
  def mime_formats, do: [:csv]

  @impl true
  def stream?, do: false

  @impl true
  def decode_eager(body) do
    NimbleCSV.RFC4180.parse_string(body)
  end
end

    Streaming decoder

    Implements all six callbacks. The state machine for a buffer-of-lines format looks like:

    def decode_start, do: ""

def decode_chunk(chunk, buffer) do
  # split off complete records; return decoded items + remaining partial buffer
  ...
end

def decode_finish(buffer) do
  # parse any trailing partial record (if non-empty)
  ...
end

    Summary

    Callbacks

    decode_chunk(binary, state)

    Decodes a chunk of bytes given the current decoder state.

    decode_eager(binary)

    Decodes the full (already-decompressed) response body and returns the decoded term.

    decode_finish(state)

    Finalizes the decoder; returns any items still pending in the trailing state.

    decode_start()

    Returns the initial decoder state (called once before streaming begins).

    mime_formats()

    Returns the MIME format keys this decoder handles.

    stream?()

    Returns true if the decoder supports :stream-mode chunk decoding.

    Callbacks

    decode_chunk(binary, state)

    (since 0.6.0) (optional) 
    @callback decode_chunk(binary(), state :: term()) :: {[item :: term()], state :: term()}

    Decodes a chunk of bytes given the current decoder state.

    Returns {decoded_items, new_state}. Items not yet completable (e.g. a partial NDJSON line that hasn't seen its newline) stay in new_state for the next call.

    decode_eager(binary)

    (since 0.6.0) 
    @callback decode_eager(binary()) :: term()

    Decodes the full (already-decompressed) response body and returns the decoded term.

    decode_finish(state)

    (since 0.6.0) (optional)

    Finalizes the decoder; returns any items still pending in the trailing state.

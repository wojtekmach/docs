    Req.Decode (req v0.6.0-dev)

    View Source

    Decodes the response body based on the content-type response header.

    Routes content-types to decoders by MIME format key:

    FormatDecoderStream?
    jsonReq.JSON✗
    ndjson, jsonlReq.NDJSON✓
    event_stream (SSE)Req.EventStream✓

    Options

    • :decoders - Configures response body decoding. A keyword list mapping MIME format keys to Req.Decoder modules.

    Set a specific key to false to disable just that decoder, or set the whole option to false to disable decoding entirely.

    • list({atom(), module() or false})
    • false

    Defaults to [json: Req.JSON, ndjson: Req.NDJSON, event_stream: Req.EventStream].

    • :stream - If set, invokes a user callback per decoded item (for streaming decoders) or per body chunk (for non-streaming decoders or unknown content-types). The callback receives (item_or_chunk, conn) and must return the updated conn.

    conn.resp_body is not touched by :stream — use conn.assigns to accumulate per-stream state.

    • &function/2
    • nil

    Defaults to nil.

    • :raw - When true, the response body is left as-is (no decompression, no decoding). boolean(). Defaults to false.

    Examples

    Override the default :json decoder for a single call — useful when a server mis-labels an NDJSON stream as application/json:

    Req.fetch!("https://httpbin.org/stream/3", decoders: [json: Req.NDJSON])

    Disable a specific decoder:

    Req.fetch!(url, decoders: [json: false])

    Disable all decoding:

    Req.fetch!(url, decoders: false)

    Stream NDJSON items as they're decoded:

    Req.fetch!("https://reqbin.org/ndjson",
  stream: fn item, conn ->
    IO.inspect(item)
    conn
  end
)
